Modely — kvalita vs sovereignty

SWE-Bench Verified score (vyšší = lepší schopnost vyřešit reálné GitHub issue). Open-weight licence = lze hostit v EU.

Klíčový insight: Open-weight špička (DeepSeek V4-Pro 80.6, Qwen3.6-Plus 78.8) je dnes blíž k Claude Opus 4.7 (87.6) než k loňskému Sonnet 4 (~70). Gap se uzavřel — open-weight nyní dostačuje pro většinu denní práce.
EU AI Act enforcement 2. 8. 2026: Modely čínského původu (Qwen, DeepSeek, GLM, Kimi) potřebují provenance review — security audit, hash verifikaci vah, air-gapped runtime. Řešitelné, ale počítat se 4-6 týdny právního review.

Rizika a mitigace

Strategická flexibilita: váhy modelů jsou portovatelné, gateway je vlastní, hybrid zaručuje graceful degradation.

  • Anthropic/OpenAI zdraží escalation tier 2-3× Snížit eskalaci v sovereign fázi, posílit GLM-4.7 / Kimi K2.6 jako "near-frontier".
  • Nebius / EU API zdraží Migrace na dedikovaný GPU rental (Scénář B) — Nebius i OVHcloud nabízejí stejnou cestu. Váhy modelů jsou portovatelné.
  • EU AI Act zakáže čínské modely Přepnout primární na Llama 4-Coder (až vyjde) + Codestral. Degradace SWE-Bench ~5-10 bodů.
  • OVHcloud / Nebius výpadek Multi-region: primary Nebius FI, secondary OVHcloud FR. Gateway routuje automaticky.
  • Vývojáři nepřijmou kvalitu Pilot odhalí dřív, než se utopí kapitál. Claude Code Max zůstává jako fallback (degradace, ne nahrazení).
  • MLOps team nestíhá Pilot = serverless, žádný GPU cluster. Scénář B+ externí kontrakt (Nebius / OVHcloud managed inference).
  • GPU trh zkolabuje, Blackwell zlevní Žádný CapEx ve Fázích 1-2 — flexibilita zachována pro 2028 refresh.
